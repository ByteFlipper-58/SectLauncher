# SectLauncher v26.0.5

SectLauncher v26.0.5 is a feature-focused update built around better world management, faster access to gameplay, a cleaner settings experience, and stronger cross-platform stability.

## Highlights

- Added a full **Worlds Manager** with search, sorting, filtering, grouping by instance, and direct world actions.
- Introduced launcher-style **Quick Play** for **worlds, servers, and Realms** directly from the home screen.
- Fully redesigned the **Settings** experience with cleaner navigation, smoother transitions, and more consistent UI behavior.
- Improved launcher reliability across **Windows, Linux, and macOS**, including runtime handling, startup flows, and packaging updates.

## New

### Worlds Manager

- Browse worlds across instances from a dedicated management page.
- Filter by instance, loader, and game mode.
- Sort by last played, name, size, or instance.
- Group worlds by instance for faster navigation in larger libraries.
- Rename, transfer, back up, restore, open, launch, and delete worlds directly from the launcher.

### Quick Play

- Added a new Quick Play rail on the home screen for recent or pinned entries.
- Supports **singleplayer worlds**, **multiplayer servers**, and **Realms**.
- Includes hover cards with contextual details and quick pin/unpin actions.
- Lets users choose between **loader icons** and **world icons**, with the preference saved in launcher settings.

## Improved

### Settings and UI

- Reworked the Settings page into a cleaner, more structured flow across **General**, **Game**, **Appearance**, **System**, and **About**.
- Improved page transitions and shell state handling for a smoother overall experience.
- Refined loading and transition behavior in key parts of the interface.
- Fixed modal layering so dialogs always stay above the main application UI.

### Launcher, Accounts, and Compatibility

- Restored **offline multiplayer** and **Realms** behavior across Minecraft versions.
- Offline accounts now get their UUID generated immediately during creation for more reliable launches.
- Improved mod dependency handling and back navigation behavior.
- Fixed instance repair validation using proper **SHA1-based file checks**.

### Platform and Packaging

- Fixed cross-platform runtime and launcher handling, especially on **macOS**.
- Hardened startup behavior and local asset handling on **macOS** and **Linux**.
- Optimized Windows release builds and fixed cached media loading issues.
- Updated the Windows installer to auto-install **WebView2** when required while staying lightweight.

## Under the Hood

- Completed a major frontend refactor around a stricter **app / features / shared** structure.
- Continued the Tauri backend refactor with clearer **application / domain / infrastructure** boundaries.
- Extracted a dedicated platform infrastructure layer to make long-term Windows, Linux, and macOS support easier to maintain.

## Upgrade Notes

- No manual migration is required for this release.
- Existing instances, worlds, and accounts should continue to work normally.
- On Windows, first-run setup is now smoother on systems where **WebView2** is not already installed.
