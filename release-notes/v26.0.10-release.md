# SectLauncher v26.0.10-release

SectLauncher v26.0.10-release is a major catalog and distribution update focused on bringing CurseForge into the launcher without breaking the existing Modrinth experience.

## Highlights

- Added full **CurseForge** support alongside **Modrinth** with a dedicated provider switch in the catalog.
- Added support for **CurseForge mods, resource packs, shaders, and modpacks**.
- Added import for **native CurseForge ZIP modpacks** with automatic format detection.
- Added fallback behavior for files that cannot be downloaded through third-party clients: the launcher now opens the official **CurseForge project page** instead of failing with a hard install error.

## New

### CurseForge Integration

- Added a provider-aware catalog flow shared by Modrinth and CurseForge.
- Added provider-specific search, filters, pagination, and cache state.
- Added provider-aware project pages, install actions, and installed-content metadata.
- Added proper support for CurseForge categories, Minecraft versions, and loader filters.

### Modpack Import

- Added native import for CurseForge ZIP manifests.
- Pack format is now auto-detected between `.mrpack` and CurseForge ZIP.
- CurseForge imports resolve the correct Minecraft version and loader before instance creation.
- Missing or unavailable CurseForge files no longer abort the whole import flow and are instead collected into a guided follow-up queue.

## Improved

### Catalog UX

- Fixed CurseForge category rendering and filter behavior.
- Fixed provider labels in logs and operation messages so CurseForge actions are no longer reported as Modrinth.
- Improved mixed-library handling for installed content coming from different providers.

### Performance

- Moved changelogs to GitHub-hosted release notes.
- Added local caching for fetched changelog entries to make the news feed faster and more resilient.
- Deferred analytics initialization and lazy-loaded heavy documents and views to reduce startup cost.
- Split provider cache scopes so Modrinth and CurseForge data no longer collide.

## Upgrade Notes

- No manual migration is required for existing Modrinth installs.
- Some CurseForge files may still require manual download if the author has disabled third-party distribution.
- In those cases, SectLauncher redirects users to the official CurseForge project page instead of bypassing platform rules.
