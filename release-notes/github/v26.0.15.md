# SectLauncher v26.0.15

## Overview

This release bundles the work shipped between **2026-04-15** and **2026-04-28**.

The main focus of this build is:

- smarter Java requirement detection from Minecraft metadata
- managed **Latest Release** and **Latest Snapshot** instance channels
- a redesigned settings flow and cleaner startup/update behavior
- improved home selection persistence and instance synchronization
- configurable sidebar hiding with contextual back buttons
- stronger app/module boundaries and smaller frontend flows
- improved Markdown/media rendering and startup window reliability

## What's Changed

### Settings, startup, and update flow

The settings area was reworked to make launcher configuration easier to scan and less dependent on one oversized settings screen.

- Redesigned the settings flow and split more settings behavior into dedicated cards and sections.
- Added a dedicated privacy/settings card and removed older Quick Play manager placement from the settings page.
- Improved startup update state handling so update checks and startup UI behavior are easier to coordinate.
- Refined splash screen behavior and layout integration around launcher startup.
- Updated release note delivery with a GitHub-oriented changelog file for the previous release.
- Expanded localization coverage for the new settings and navigation options.

Relevant commits:
- `8a7d528` Redesign settings flow and startup update behavior
- `81cb6e5` Fix startup, media embeds, and instance sync

### Java requirements and instance setup

Instance setup now relies more directly on Minecraft metadata instead of local hardcoded assumptions.

- Added backend support for resolving Java requirements from Minecraft version metadata.
- Added frontend API and type coverage for Java requirement resolution.
- Reworked instance creation and installation paths to use resolved Java requirements.
- Removed the older frontend-only Java runtime resolver path.
- Improved Java compatibility presentation in instance forms and Java settings.
- Preserved instance-specific Java and launch settings when editing existing instances.

Relevant commits:
- `abcd68f` Resolve Java requirements from Minecraft metadata
- `e6ec90e` Add managed latest release and snapshot channels

### Managed latest release and snapshot channels

The home screen and instance creation flow now support managed targets for the latest stable and snapshot Minecraft versions.

- Added managed **Latest Release** and **Latest Snapshot** channels.
- Added launcher settings support for remembering managed target UUIDs.
- Added a snapshot visual asset and home target presentation helpers.
- Updated the home instance selector and play button behavior for managed targets.
- Added managed target handling to instance creation, installation, and selection persistence.
- Improved the instance grid and form behavior around managed target names and versions.
- Fixed a home instance persistence loop that could repeatedly resync selected targets.

Relevant commits:
- `e6ec90e` Add managed latest release and snapshot channels
- `365099a` Fix home instance persistence loop

### Home, sidebar, and page navigation

Navigation behavior is now more flexible for layouts where the sidebar should stay out of the way.

- Added a user setting for hiding the sidebar on selected pages.
- Added per-page sidebar hiding support across Home, Instances, Mods, News, Settings, and Worlds.
- Added a shared page back button component.
- Back buttons are now shown only when the sidebar is hidden, avoiding duplicate navigation controls.
- Moved sidebar hidden page state into the app state layer during cleanup.
- Localized the sidebar hiding setting across supported languages.

Relevant commits:
- `6d2b259` Add configurable sidebar hiding
- `7875838` Show back buttons only when sidebar is hidden
- `821ce43` Clean up dead frontend exports

### Startup, media embeds, and instance sync fixes

This release includes several reliability fixes around startup, media rendering, and synchronized instance state.

- Improved splash screen command handling and startup window bootstrap behavior.
- Adjusted app/provider initialization order for system, runtime, theme, EULA, and launcher update flows.
- Improved selected instance persistence and managed target synchronization.
- Improved Markdown rendering for embedded media and external content layouts.
- Added remote image fallback handling for cached images.
- Cleaned up modal/detail behavior in mods and refreshed related controller logic.
- Tightened frontend lint boundary rules around app and feature imports.

Relevant commits:
- `81cb6e5` Fix startup, media embeds, and instance sync

### Architecture and flow cleanup

The codebase received a larger cleanup pass to keep feature logic smaller and enforce clearer boundaries between app, feature, shared, application, domain, and infrastructure layers.

- Added architecture/import boundary rules to ESLint and documented the intended layering.
- Split large Quick Play UI into smaller presentation and modal components.
- Split custom and managed instance installation into dedicated model modules.
- Added a shared install progress listener for instance install state.
- Split mods catalog import, local modpack import, install plan execution, task info, and catalog helper logic out of larger hooks.
- Split mod detail action/sidebar UI into focused components.
- Introduced service/port boundaries for system and worlds application logic on the Rust side.
- Refactored auth, downloader, launcher, worlds, and system code toward clearer application/infrastructure contracts.
- Removed dead frontend exports and legacy barrel paths that were no longer part of the current architecture.
- Reduced large frontend modules again by extracting legal document loading and mrpack export modal sections.

Relevant commits:
- `c043fe2` refactor: enforce app boundaries and split large UI flows
- `821ce43` Clean up dead frontend exports
- `f89b403` refactor: split large frontend modules

### Mods, Quick Play, worlds, and shared runtime behavior

Several feature areas were reorganized so long-running actions and cross-feature behavior are easier to maintain.

- Moved Quick Play presentation logic out of the main home section.
- Added a dedicated Quick Play settings modal, button, and tooltip components.
- Split mods install/import flows into focused modules for catalog modpacks, local modpacks, dependency planning, and task metadata.
- Improved mods detail action/sidebar composition.
- Added worlds application service and port boundaries.
- Adjusted quick play entries, launches, servers, and shared world logic to fit the new service shape.
- Updated shared Tauri command typings and shared type exports after the cleanup.

Relevant commits:
- `c043fe2` refactor: enforce app boundaries and split large UI flows
- `821ce43` Clean up dead frontend exports

### Versioning

- Updated application versioning to **26.0.15** across the frontend package metadata, Tauri config, Rust manifest, and lockfiles.

## Upgrade Notes

- Existing instances keep their current Java, memory, JVM, resolution, and fullscreen settings.
- Managed latest release/snapshot targets may create or reuse launcher-managed instances depending on the saved target UUIDs.
- Sidebar hiding is opt-in through settings and does not change page navigation unless enabled.
- The refactors are intended to preserve user-facing behavior while making future fixes and feature work easier to ship.
