# SectLauncher v26.0.11

## Overview

This release bundles the work shipped between **2026-04-02** and **2026-04-04**.

The main focus of this build is:

- smarter instance setup and Java/runtime selection
- new instance performance profiles
- improved Quick Play management
- more reliable Microsoft/account flows
- clearer mods catalog behavior
- a new bottom background tasks bar for long-running operations

## What's Changed

### Instance setup, Java, and launcher runtime

Instance creation and repair flows were reworked to make runtime selection more predictable and more self-sufficient.

- Refactored Java runtime management across the launcher stack.
- Improved Java compatibility detection for instance creation and installation based on Minecraft version and loader.
- Added stronger fallback handling so the launcher can reuse or download a suitable Java runtime when the currently configured one is not compatible.
- Expanded Java management behavior in settings, including cleaner installed/downloadable runtime handling.
- Unified loader version selection logic for supported loaders, reducing mismatch cases between creation and repair flows.

Relevant commits:
- `791d6b6` Refactor Java runtime management
- `10d9a86` Improve Java runtime handling and changelog delivery
- `bbdf9e0` Unify loader version strategies

### Instance performance profiles

This release adds a more opinionated performance layer for instances so users do not have to manually tune memory and JVM flags every time.

- Added default performance profiles: **Auto**, **Light**, **Balanced**, **Heavy**, and **Custom**.
- Added automatic recommended memory/JVM behavior for the **Auto** profile.
- Preserved per-instance manual overrides when users intentionally deviate from the recommended setup.
- Integrated the profile system into instance creation and related settings flows.

Relevant commits:
- `a5f99a7` Add instance performance profiles

### Quick Play and home interactions

Quick Play management and entry behavior were refined to make the home experience more controllable and easier to maintain from settings.

- Improved Quick Play interaction behavior on the home screen.
- Added Quick Play management controls in settings.
- Improved backend handling for quick world-related entries.

Relevant commits:
- `91f3b15` Refine quick play management and interactions

### Accounts and authentication

Account flows received both reliability fixes and UI cleanup, especially around Microsoft sessions and sidebar account presentation.

- Fixed the Microsoft token refresh flow to improve session persistence.
- Refactored account avatar and sidebar account UI into dedicated feature-level components.
- Improved account loading, switching, deleting, and add-account states.
- Added clearer user-facing error handling for account actions.
- Reduced broken avatar / broken player-head cases in the sidebar.

Relevant commits:
- `71c9b08` Fix Microsoft token refresh flow
- `54d36cb` Refactor sidebar account avatars
- `5314b49` Improve accounts UX states

### Mods catalog behavior

Catalog result handling was improved so “no results” states are more diagnostic and less generic.

- Improved mods catalog empty-state behavior.
- Differentiated between filter mismatch, compatibility mismatch, and provider-related no-result states instead of relying on one fallback message.

Relevant commits:
- `aa3be97` Improve mods empty states

### Background tasks and long-running operations

Long-running launcher operations now have a dedicated runtime UI instead of relying only on scattered local loading states.

- Generalized the install queue into a broader background task model.
- Added a new bottom background tasks bar in the content area.
- Added richer task state handling including progress, success, error, retry actions, and task-specific navigation targets.
- Wired long-running instance and mods operations into the task system, including install, import, export, repair, and update flows.
- Added animation and interaction polish for the tasks bar, including staged appearance, upward details expansion, staggered task entry, outside-click close, and keyboard dismissal.

Relevant commits:
- `f1c4a3e` Release 26.0.11 UX polish

### Worlds cleanup

Some older worlds-related automation and integrity features were removed as part of cleanup across the worlds/settings stack.

- Removed legacy world auto-backup automation paths.
- Removed legacy world integrity-check flows that were no longer being carried forward in the current UX direction.

Relevant commits:
- `b5f84d4` Remove world backup automation and integrity checks

### Release notes and versioning

- Added resources/footer updates for release notes and related manifest cleanup.
- Updated application versioning to **26.0.11** across the frontend, Tauri config, and Rust manifests.

Relevant commits:
- `a4318b7` Add resources footer to release notes, update manifest
- `f1c4a3e` Release 26.0.11 UX polish
