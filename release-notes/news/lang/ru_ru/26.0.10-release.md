## SectLauncher 26.0.10-release — CurseForge и крупное обновление каталога

### CurseForge теперь встроен в лаунчер
* Добавлена полноценная поддержка **CurseForge** как второго провайдера каталога наряду с Modrinth.
* В магазине появился явный **переключатель провайдера** между **Modrinth** и **CurseForge** без смешивания выдачи.
* CurseForge теперь работает не только для **модов**, но и для **ресурс-паков**, **шейдеров** и **модпаков**.
* Для каждого провайдера отдельно сохраняются поиск, фильтры, пагинация и кэш, поэтому переключение между источниками стало предсказуемым и быстрым.

### Установка контента и карточки проектов
* Моды, ресурс-паки и шейдеры из **CurseForge** теперь можно устанавливать прямо из лаунчера, если файл доступен через API.
* Карточки проектов и страницы деталей стали **provider-aware**: отображают источник, корректные ссылки и правильные действия для Modrinth и CurseForge.
* Если конкретный файл нельзя скачать через сторонний клиент из-за настроек автора или ограничений CurseForge, лаунчер больше не падает с ошибкой:
  * вместо этого показывается предупреждение;
  * кнопка установки открывает **официальную страницу проекта на CurseForge**.
* Установленный контент теперь хранит **метаданные провайдера**, поэтому обновление, удаление, просмотр деталей и работа смешанных библиотек стали надёжнее.

### Импорт сборок из CurseForge
* Добавлен импорт **нативных CurseForge ZIP-сборок** вместе с уже существующей поддержкой `.mrpack`.
* Формат сборки теперь определяется автоматически, поэтому импорт идёт через правильный поток без ручного выбора источника.
* Лаунчер читает CurseForge manifest, определяет версию Minecraft и загрузчик и создаёт инстанс с корректной конфигурацией.
* Если часть файлов из CurseForge недоступна, импорт больше не валит весь процесс:
  * инстанс всё равно создаётся;
  * проблемные файлы попадают в **guided queue** для ручной догрузки;
  * в интерфейсе и логах показывается предупреждение вместо жёсткого fail-state.

### Фильтры, категории и UX каталога
* Полностью переработан backend каталога: теперь он работает через **единый provider-neutral слой**, а Modrinth и CurseForge подключаются как отдельные адаптеры.
* Исправлена работа **категорий, версий Minecraft и фильтров загрузчиков** для CurseForge.
* Исправлены проблемы с отображением категорий и иконок в фильтрах CurseForge, включая кейсы с **resource packs** и **shaders**.
* Логи и уведомления теперь показывают **реальный источник операции**, поэтому при установке и импорте больше не появляется ложная пометка `Modrinth` для CurseForge-контента.

### Производительность и инфраструктура
* Чейнджлоги вынесены в **GitHub-hosted release notes**, а экран новостей теперь кэширует записи локально, чтобы они открывались быстрее и переживали временные проблемы с сетью.
* Уменьшена стартовая нагрузка лаунчера:
  * тяжёлые экраны и документы подгружаются лениво;
  * аналитика инициализируется позже;
  * часть контента больше не раздувает стартовый bundle.
* Modrinth и CurseForge теперь используют **раздельные cache scopes**, поэтому данные провайдеров не смешиваются между собой.

### Важное для пользователей
* Ручная миграция не требуется: старые установки Modrinth продолжают распознаваться и работать.
* Часть проектов CurseForge может требовать ручного скачивания с сайта, если автор ограничил внешнее распространение файлов.
* В таких случаях SectLauncher направляет пользователя на официальную страницу проекта, а не пытается обойти ограничения.
