## SectLauncher 26.0.5 — What's New

### 🌍 Worlds and Quick Play
* Added a full **Worlds Manager** with search, sorting, filters by instance, loader, and game mode, plus one-click world launch.
* Worlds can now be **grouped by instance**, searched by name, and inspected with extra metadata such as last played time and size.
* Worlds can now be **renamed, transferred between instances, backed up, restored, and deleted** directly from the launcher.
* The home screen now includes launcher-style **Quick Play** for **worlds, servers, and Realms** with pinning support, hover cards, and configurable icon mode.
* Quick Play can display either **loader icons** or **world icons**, and that preference is saved in settings.

### ⚙️ Settings and Interface
* The **Settings page** has been fully redesigned with cleaner navigation, better section structure, and smoother category transitions.
* The **General / Game / Appearance / System / About** sections now feel like one consistent control surface instead of separate screens.
* Improved launcher shell state handling and page transition behavior across the UI, including better skeleton/loading presentation.
* All modal windows now stay **above the app layer** and animate more cleanly when opening and closing, without conflicting with the sidebar or other UI layers.

### 🧩 Launcher and Compatibility
* Restored **offline multiplayer** and **Realms** behavior across different Minecraft versions.
* Offline accounts now get their UUID generated immediately on creation, making launches more reliable and avoiding temporary profile mismatches.
* Fixed instance **Repair** validation by switching to proper **SHA1-based checks**, improving damaged or missing file detection.
* Improved mod dependency handling, mod detail behavior, and back navigation flow inside the launcher.

### 💻 Platform Stability
* Fixed cross-platform **runtime/launcher** handling, with special attention to **macOS**.
* Hardened window startup and local asset loading on **macOS/Linux**, including splash/startup flows.
* Optimized Windows release builds, improved packaging behavior, and fixed cached media loading issues.
* The Windows installer can now auto-install **WebView2** while keeping the installer lightweight and reducing first-run setup issues on clean systems.

### ⚙️ Under the Hood
* Completed a major frontend and Tauri backend refactor with stricter internal layering.
* The frontend is now organized around **app / features / shared**, while the Tauri backend follows clearer **application / domain / infrastructure** boundaries.
* Added a dedicated platform infrastructure layer to simplify long-term support for Windows, Linux, and macOS.
