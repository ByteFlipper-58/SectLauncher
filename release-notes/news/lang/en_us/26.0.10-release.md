## SectLauncher 26.0.10-release — CurseForge and a major catalog update

### CurseForge is now built into the launcher
* Added full **CurseForge** support as a second catalog provider alongside Modrinth.
* The shop now includes an explicit **provider switch** between **Modrinth** and **CurseForge** instead of mixing results.
* CurseForge now works not only for **mods**, but also for **resource packs**, **shaders**, and **modpacks**.
* Search state, filters, pagination, and cache are now stored per provider, so switching between sources is faster and more predictable.

### Installing content and project pages
* Mods, resource packs, and shaders from **CurseForge** can now be installed directly inside the launcher when the file is available through the API.
* Project cards and detail pages are now **provider-aware**, showing the correct source, links, and actions for both Modrinth and CurseForge.
* If a specific file cannot be downloaded through a third-party client because of author settings or CurseForge restrictions, the launcher no longer fails with a hard install error:
  * it shows a warning instead;
  * the install action opens the **official CurseForge project page**.
* Installed content now stores **provider-aware metadata**, making updates, removal, details, and mixed libraries more reliable.

### CurseForge modpack import
* Added support for importing **native CurseForge ZIP modpacks** alongside existing `.mrpack` support.
* The launcher now auto-detects the pack format and routes import through the correct provider flow without asking the user to choose manually.
* CurseForge manifests are parsed to resolve the correct Minecraft version and loader before creating the instance.
* If some CurseForge files are unavailable, the import no longer fails as a whole:
  * the instance is still created;
  * missing files are moved into a **guided queue** for manual follow-up;
  * the UI shows a warning instead of a full failure state.

### Filters, categories, and catalog UX
* The catalog backend has been reworked around a **provider-neutral layer**, with Modrinth and CurseForge connected as separate adapters.
* Fixed **category, Minecraft version, and loader filter** behavior for CurseForge.
* Fixed CurseForge category and icon rendering issues, including problem cases for **resource packs** and **shaders**.
* Logs and user-facing operation messages now show the **actual provider**, so CurseForge flows no longer appear as `Modrinth` in the UI.

### Performance and infrastructure
* Changelogs have been moved to **GitHub-hosted release notes**, and the news screen now caches fetched entries locally for faster loading and better resilience during temporary network issues.
* Reduced launcher startup cost:
  * heavy screens and legal documents are lazy-loaded;
  * analytics initializes later;
  * some content no longer inflates the initial app bundle.
* Modrinth and CurseForge now use **separate cache scopes**, so provider data stays isolated.

### Important notes
* No manual migration is required: existing Modrinth installs are still recognized and remain compatible.
* Some CurseForge projects may still require manual download from the website if the author has restricted third-party file distribution.
* In those cases, SectLauncher opens the official project page instead of trying to bypass platform rules.
